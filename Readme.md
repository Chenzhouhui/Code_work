## 运算符
() [] -> ! ~ ++ -- &* +- << >> == >< &|^ && || ?: = += -=
- 优先级：简单记就是：!>算数运算符>关系运算符>&&>||>赋值运算符
- 单目优于双目
- 单目、条件、赋值是右左结合
- ! 表示取非，对于整形变量，只要不为0，使用!取反结果都为0，类似bool
- ~ 取反，代表位的取反，对于整形变量，对每个二进制位进行取反，0变1，1变0
- ? : 三目运算符
- 源码、反码、补码
 - 源码：为二进制数
 - 反码：正数的反码与源码相同，负数的反码0变1，1变0（符号位不变）1000 1010 ->1111 0101
 - 补码：正数的补码与源码相同，负数的补码为反码+1，eg: 1000 1010 -> 1111 0110
- 短路求值
```
int a = 5, b = 3;
    !a && (b++);
    printf("%d, %d\n", a, b); //5,3
```    
## 一、关键字
### 1.1 Continue
- 跳过本次循环中余下未执行的语句，立即进行下一次的循环条件判定，可以理解为仅结束本次循环，注意continue关键字并没有终止整个循环**只能用在循环语句**
### 1.2 Break
- 在while、for循环中跳出本层循环
- 在switch-case中跳出对应的case
- 如果for(){ switch case: break },for循环没有影响
```
for(...)
{
    switch(1)
    {
        case 1: 
            printf("hello\n");
            break;
    }
}
```
- 如果是下面这种情况：就直接退出循环
```
while(1)
{
    switch(1)
    {
        case 1: 
            printf("hello\n");
            break;
    }
}
```
- 在switch-case语句中，如果没有break，那么就会直接执行其它case下面的语句
- break语句用在循环体中，结束本层循环；continue语句结束本次循环；**break用于循环语句或者开关语句**
### 1.3 Return
- 结束当前循环，退出函数，用于函数体中，返回特定值。
### 1.4 Goto
- 无条件跳转
### 1.5 Volatile（编译优化阶段）
- 作用：告诉编译器该变量是容易发生变化的，不能对该变量进行优化，每次取值都必须从内存中取值，而不是直接去取之前在寄存器中的值。
```
volatile int a = 20, b, c;
b = a; //先从a的内存取值放到寄存器，再把寄存器中的值存给b
c = a; //把寄存器的值存到c内存
```
以上看出编译器对c=a这一步进行了优化，不再从内存中取值，而是直接读取寄存器，如果这段时间内a的值发生变化，那么c就不能得到最新的值，这个时候就需要用volatile告诉编译器，不要对变量a优化，每次都从内存取值。
- 常见场景：
 - (1)多线程使用共享变量：因为多线程是多核操作，同时进行。
 - (2)中断:中断程序会修改其他程序中使用的变量
 - (3)硬件寄存器：因为寄存器随时会被修改，好比AD转换器的寄存器，随时会因为外部电压的变化而变化
 - (4)外部任务会修改变量
### 1.6 struct(结构体)(C和C++区别，求结构体大小，使用的注意事项)
- 在c语言结构体中不允许有函数，而c++可以
- c语言结构体不可以继承，c++中可以继承
- c语言结构体的使用必须要用别名或者struct，不能直接使用例如：
```
Struct student
{
    int age;
    int num;
}
Typedef struct student student;//必须要别名才可以使用
---------
struct student student;//或者在使用的时候加上struct
```
- 访问权限不同：在c中默认是共有的，不可以修改权限，在c++中权限可以修改。
- 初始化：在c中不可以初始化数据成员，c++中可以初始化。
- **c++中空结构体大小为1，c为0**
### 1.7 class和struct的区别？
- 继承权限：class默认是private，struct是public
- 访问权限：class作为对象的实现体，默认是私有访问：而struct作为数据结构的实现体，是共有访问的
- class可以用于定义模板，而struct不能

### 1.8 Union(联合体)
- (1)union和struct的区别
 - 对于联合体所有的成员都共享一块内存，而结构体是所有变量内存的叠加，需要考虑字节对齐问题，对于联合体来说，只要修改里面的成员的数据就会修改其他成员的数据，而结构体成员数据是不影响的。
- (2)联合体一般用来判断大小端问题：
```
union my
{
    short t;
    char b[2];
}
Typedef union my My;
int main()
{
    My test;
    test.t = 0x0102;
    if(test.b[0] == 0x01 && test.b[1] == 0x02)
    {
        printf("这是大端字节序\n");//高字节放在低地址，低字节放在高地址
    }
    if(test.b[0] == 0x02 && test.b[1] == 0x01)
    {
        printf("这是小端字节序\n");//低字节仿真低地址，高字节放在高地址
    }
} 
```
- (3)大小端转换问题
```
int swap(int value)
{
    value = ((value & 0x000000ff) << 24) |
            ((value & 0x0000ff00) << 8) |
            ((value & 0x00ff0000) >> 8) |
            ((value & 0xff000000) >> 24) |
            reture value;
}
```
- (4)计算占用空间的大小
计算的时候需要考虑字节对齐：所占空间必须是成员变量字节最大的整数倍；每个变量类型的偏移量必须是该变量类型的整数倍；对于联合体，由于所有变量都是共用一块内存，还需注意数组占用的最大内存。
```
//联合体的计算：
//最终的大小得是结构体中类型的最大长度的整数倍，并且能容下所有类型
union data
{
    char a;
    double b[3];
    char c;
};
typedef union data DATA;
//结构体的计算：
//最终的大小得是结构体中类型的最大长度的整数倍
//除了结构体中的联合体外，其他类型的偏移量必须是该类型的整数倍
//如果里面有联合体，该联合体的起始位置要满足该联合体里的最大长度类型的偏移量
struct test 
{
    char a; //1
    DATA d;//8+24
    char b;//33
    char c;/34
};//最终40
typedef struct test TEST;
int main()
{
    printf("%ld\n", sizeof(DATA)); //24
    printf("%ld\n", sizeof(TEST)); //40
    return 0;
}
```
### 1.9 Enum 枚举
- **里面的变量会自加**，在枚举中，某个枚举变量的值默认为前一个变量的值+1，而如果第一个枚举变量没有被赋值，则其默认值为0

### 1.10 Typedef
- #define和typedef的区别 （宏，关键字，预处理，编译，检查）
 - #define是c语言中定义的语法，**是预处理指令**，在预处理时进行简单而机械的字符串替换，**不做正确性检查**，只有在编译已被展开的源程序时才会发现可能的错误并报错。
 - typedef是**关键字**，在编译处理时，**有类型检查功能**。它在自己的作用域内给一个已经存在的类型一个别名，但是不能在一个函数定义里面使用typedef。用typedef定义数组、指针、结构体等类型会很方便，增强可读性。
- 注意#define和typedef定义指针的区别
```
#define myptr int* p;
myptr a, b; //a是int* a,  而b时int b；
typedef int* myptr;
myptr a, b; //a是int* a， b也是int* b；
```

### 1.11 Const (C语言中的变量：局部，函数形参，返回值，指针，typedef的区别，数组大小定义，case；C++中：常函数，常对象)
- 定义变量，表示该变量是常量，不允许修改 //const int a = 1；
- 修饰函数参数，表示函数体内不能修改这个变量的值
- 修饰函数的返回值，const char getstr()
- const修饰的常量
 - const int a;int const a;是一样的，都是修饰变量a为常量
 - const * int a;int const *a;修饰的这个指针指向的内容不允许修改，这个指针的地址可以改变**常量指针**
 - int * const a;修饰的指针，表示这个指针的地址不能改变，地址对应的内容可以修改**指针常量**
 const int * const a;表示地址不能改，地址内的内容也不能改
- const修饰函数在c++的类中是不能virtua虚函数
- 假设const n = 3; int num[n] = {1, 2, 3};这个错误的。在c语言中n并不是真正意义上的常量，还是变量，对于数组来说，必须开辟一个正确的数组大小，在c++中是可以编译过去的。
  类似的，在switch case n:也是错误的
- 在c++类中有常函数的概念，比如在类中定义如下函数：void fun() const { }，像这种就是常函数，只能读取类中数据，不能修改
- 常对象，常对象只能调用常函数
- const修饰的变量存放位置
 - 对于const修饰的局部变量：存放在栈中，代码结束后就会释放，在c语言中可以通过指针来修改里面的值
 - 对于const修饰的全局变量：（已经初始化的）存放在**只读数据段(.rodata)**，不可以通过指针修改里面的值，未初始化的放在.bss

### 1.12 extern（连接阶段）
- 声明外部变量(保存在数据段)
  - 在文件a.c定义和声明变量a,int a= 20;//这里会建立存储空间
  - 通过extern在b.c文件里面声明a之后就可以使用了，不能初始化eg：extern int a;
该作用是告诉编译器我在其他文件定义了变量a并且分配了空间，不再为该变量申请空间
- 对函数：声明外部函数（可跨文件调用）
- extern "C" 禁止名字粉碎，实现 C/C++ 混合编译
```
#ifdef __cplusplus
extern "C" {
#endif

void func(void); // 按 C 规则编译，不重载改名

#ifdef __cplusplus
}
#endif
```

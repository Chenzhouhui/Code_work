### 1.5 Volatile（编译优化阶段）
- 作用：告诉编译器该变量是容易发生变化的，不能对该变量进行优化，每次取值都必须从内存中取值，而不是直接去取之前在寄存器中的值。
```
volatile int a = 20, b, c;
b = a; //先从a的内存取值放到寄存器，再把寄存器中的值存给b
c = a; //把寄存器的值存到c内存
```
以上看出编译器对c=a这一步进行了优化，不再从内存中取值，而是直接读取寄存器，如果这段时间内a的值发生变化，那么c就不能得到最新的值，这个时候就需要用volatile告诉编译器，不要对变量a优化，每次都从内存取值。
- 常见场景：
  - (1)多线程使用共享变量：因为多线程是多核操作，同时进行。
  - (2)中断:中断程序会修改其他程序中使用的变量
  - (3)硬件寄存器：因为寄存器随时会被修改，好比AD转换器的寄存器，随时会因为外部电压的变化而变化
  - (4)外部任务会修改变量

### 1.8 Union(联合体)
- (1)union和struct的区别
  - 对于联合体所有的成员都共享一块内存，而结构体是所有变量内存的叠加，需要考虑字节对齐问题，对于联合体来说，只要修改里面的成员的数据就会修改其他成员的数据，而结构体成员数据是不影响的。
- (2)联合体一般用来判断大小端问题：
```
union my
{
    short t;
    char b[2];
}
Typedef union my My;
int main()
{
    My test;
    test.t = 0x0102;
    if(test.b[0] == 0x01 && test.b[1] == 0x02)
    {
        printf("这是大端字节序\n");//高字节放在低地址，低字节放在高地址
    }
    if(test.b[0] == 0x02 && test.b[1] == 0x01)
    {
        printf("这是小端字节序\n");//低字节仿真低地址，高字节放在高地址
    }
} 
```
- (3)大小端转换问题
```
int swap(int value)
{
    value = ((value & 0x000000ff) << 24) |
            ((value & 0x0000ff00) << 8) |
            ((value & 0x00ff0000) >> 8) |
            ((value & 0xff000000) >> 24) |
            reture value;
}
```
- (4)计算占用空间的大小
计算的时候需要考虑字节对齐：所占空间必须是成员变量字节最大的整数倍；每个变量类型的偏移量必须是该变量类型的整数倍；对于联合体，由于所有变量都是共用一块内存，还需注意数组占用的最大内存。
```
//联合体的计算：
//最终的大小得是结构体中类型的最大长度的整数倍，并且能容下所有类型
union data
{
    char a;
    double b[3];
    char c;
};
typedef union data DATA;
//结构体的计算：
//最终的大小得是结构体中类型的最大长度的整数倍
//除了结构体中的联合体外，其他类型的偏移量必须是该类型的整数倍
//如果里面有联合体，该联合体的起始位置要满足该联合体里的最大长度类型的偏移量
struct test 
{
    char a; //1
    DATA d;//8+24
    char b;//33
    char c;/34
};//最终40
typedef struct test TEST;
int main()
{
    printf("%ld\n", sizeof(DATA)); //24
    printf("%ld\n", sizeof(TEST)); //40
    return 0;
}
```

### 1.15 static(C语言：变量、函数；C++：类中变量、类中静态成员、函数)
- 定义变量
  - 静态全局变量：作用域只能作用与本文件，每次函数调用该变量都会被初始化
  - 静态全局变量：生命周期不会随函数结束而结束，直到程序结束，都是函数我们不能使用该变量，只能在函数内使用，**该变量是有记忆的**，会记住上一次的值。**该变量只会被初始化一次
    对于这2种变量来说，如果初始化的，会在数据段内；未初始化的在.bss段或者初始化为0，这2种变量都会在程序结束时才释放，只不过作用域不同。
```
int getdata()
{
    static int a  = 10;
    a++;
    printf("%d\n", a);
    return a;
}
int main()
{
    for(int i = 0; i <10; i ++)
    {
        getdata();
    }
    return 0;
}//输出11 12 13 ... 20，如果没加static，则为一直为11
```
- 定义函数
在函数返回类型前加上static，函数即被定义为静态函数。函数只能在本源文件中使用，**也就是说在其他源文件中可以定义名字一样的函数**
- 定义类中的静态成员变量(**不能再类里面初始化**，不占用类内存空间，必须定义才能使用)
在类中的静态成员变量它既可以被当作全局变量那样存储，又被隐藏于类中，类中的静态成员变量都有一块独立的存储空间，不会占用类中的空间。所有的对象都共享该静态成员，也就是说，只要有对象改变了这个值，那么其他对象就会受到影响，该数据可以使用this，也可以类中其他函数访问
- 定义类中的静态成员函数(只能访问静态成员变量)
静态成员函数也是类的一部分，而不是对象的一部分。所有这些对象的静态数据成员都共享这一块静态存储空间。注意:静态成员函数不属于任何一个对象，因此**c++规定静态成员函数没有this指针**。既然它没有指向某一个对象，也就无法对一个对象的非静态成员进行访问，即不能再静态函数里所有this指针

## 二、内存篇
### 2.1 内存分配有几种分配方式
- 栈上分配：在执行函数之前，函数内部的变量都可以在栈上创建，函数执行完毕后会自动释放
- 静态全局存取区：全局变量、静态局部变量，程序整个运行期间都存在
- 堆上分配：由程序员分配，手动申请手动释放，编译器不管，比如malloc，free
### 2.2 堆和栈有什么区别(申请方式、效率，方向)
- 申请方式不同：栈是由操作系统自由分配和释放，堆是由程序员手动申请释放
- 申请大小的限制：
栈是向低地址申请的空间，一块连续的内存，也就是说，栈的大小和地址是由系统预先规定好的，如果申请的空间超过栈的剩余空间就会提示overflow
堆的申请是向高地址扩张的，是不连续的内存区域，这是由于系统使用链表来存储的空闲内存地址
- 申请的效率
栈使用的是一级缓存，它们通常都是被调用时处于存储空间中，调用完毕立即释放；堆则是存放在二级缓存中，速度要慢些

- 堆栈溢出一般是什么原因导致的？**(递归、动态申请内存、数组访问越界、指针非法访问)**
  - 函数调用层次太深。函数递归时，系统要在栈中不断保存函数调用时的现场和产生的变量，如果递归调用太深，就会造成栈溢出，这是递归无法返回。再有，当函数调用层次过深时也可能导致栈无法容纳这些调用的返回地址而造成栈溢出。
  - 动态申请空间使用后没有释放，由于c语言中没有垃圾资源自动回收机制，因此需要程序主动释放已经不再使用的动态地址空间。申请的动态空间使用的是堆空间，动态空间使用不当造成堆溢出。
  - 数组越界访问。c语言没有提供数组下标越界检查，如果在程序中出现数组下标访问超出数组范围，在运行过程中可能会内存访问错误。
  - 指针非法访问。指针保存了一个非法的地址，通过这样的指针访问所指向的地址时会产生内存访问错误。

### 2.3 栈在c语言中有什么作用
- 用来保存临时变量，临时变量包括函数参数、函数内部定义的临时变量
- 多线程编程的基础就是栈，每个线程都拥有独立私有栈，用于存储自身局部变量、函数参数与调用栈帧，是多线程能够互不干扰、独立运行的核心基础。

### 2.4 C++的内存管理是怎样的？
在C++中虚拟内存分为代码段、数据段、bss段、堆、共享区、栈
- 代码段：包括只读存储区和文本区，其中只读存储区存储字符串常量，文本区存储程序的机械代码
- 数据段：全局变量、静态变量（全局、局部）
- bss段：未初始化的全局变量和静态变量（全局、局部），以及所有被初始化为0的全局变量和静态变量
- 堆：调用new/malloc申请的内存空间，**地址由低地址向高地址扩张**
- 映射区：存储动态链接库以及调用mmap函数的文件映射
- 栈：局部变量、函数的返回值、函数的参数，**地址由高地址向低地址扩张**

### 2.5 什么是内存泄漏？
简单来说就是申请了内存，不使用之后没有释放内存，或者说指向申请内存的指针突然又指向别的地方，导致找不到申请的内存。
有什么影响？随着程序运行时间越长、占用内存越多，最终用完内存导致系统崩溃
```
void fun()
{
    char *p = (char*)malloc(sizeof(int) * 25);
    if(p == NULL)
    {
        return -1;
    }
}
-------------------------------------------------
char *p = (char*)malloc(sizeof(int) * 25);
if(p == NULL)
{
    return -1;
}
printf("%p\n", p);
p = "hello";
printf("hello %p\n", p);
```

### 2.6 如何判断内存泄漏（减少内存泄漏）
- 良好的编码习惯，使用内存分配的函数，一旦使用完毕后要使用对应的函数释放掉
- 将分配内存的指针以链表的形式自行管理，使用之后从链表中删除，程序结束时可以检查修改链表
- 使用智能指针（c++）
- 使用常见插件，ccmalloc

### 2.7 字节对齐问题
- 什么是字节对齐
需要各种数据类型按照一定的规则在空间上排列，而不是顺序的一个接一个排放，这就是对齐。常见的就是求复合类型大小，比如结构体、联合体。
- **为什么要字节对齐**
需要字节对齐的根本原因在于CPU访问数据的效率问题
比如：如果0x02~0x05存了一个int，读取这个int需要先读0x01~0x04，留下0x02~0x04的内容，再读0x05~0x08，留下0x05的内容，这2部分拼接起来才能得到int的值，这样子读一个int就要访问2次内存，效率就低了。

### 2.8 C语言函数参数压栈顺序是怎么样的
**从左往右，并且内存中栈是由高向低扩展，所以先入栈的是最右边并且地址是高位，比如printf()函数，也都是先打印最右边**

### 2.9 C++如何处理返回值
- 基础类型（int、指针等）：通过 CPU 寄存器（EAX/RAX） 直接返回。
- 对象 / 结构体：由调用方预先分配栈空间，隐式传入地址，函数直接在目标地址构造对象，避免拷贝。
- 核心优化：支持 RVO（返回值优化） 和 NRVO（具名返回值优化），直接省略临时对象、完全消除拷贝。
- C++11 后：无优化场景下，自动使用移动语义代替拷贝，提高返回效率。
**C++ 简单值用寄存器返回，对象靠隐式地址传递 + RVO/NRVO 零拷贝返回，C++11 用移动语义进一步优化**

### 2.10 栈的空间值最大是多少？
栈大小由系统 / 编译器 / 配置决定，Windows 默认 1MB、Linux 默认 8-10MB、STM32 默认 1KB，最大可设仅受对应环境内存限制。

### 2.11 在1G内存的计算机中能否malloc(1.2)？为什么？
有可能。因为malloc是向进程中申请虚拟内存，与物理地址空间没有直接联系。

### 2.12 Stract,strncat,strcmp,strcpy哪些函数会导致内存泄漏？为什么
- strcpy：不检查长度，源字符串比目标大 → 直接越界写
- strcat：拼接时不检查剩余空间 → 越界写
- strncat：指定长度，正确使用则安全
- strcmp：只读不写，完全安全，不会溢出
这 4 个函数都不会造成内存泄漏。strcpy、strcat 会导致缓冲区溢出（无长度检查，越界写内存）。strncat 有限制长度，相对安全；strcmp 只读，无任何风险。

### 2.13 malloc,calloc,realloc内存申请函数
三者都是堆内存动态分配函数，头文件<stdlib.h>，都需要 free()释放，失败返回NULL。
```
void* malloc(size_t size);//申请size字节堆内存,不初始化内存
void* calloc(size_t num, size_t size);//申请 num 个、每个大小 size 的连续内存，全部清零
void* realloc(void* ptr, size_t new_size);//调整已分配内存的大小（扩容 / 缩容），原内存内容保留
新增内存不初始化，ptr == NULL ⇨ 等价 malloc，new_size == 0 ⇨ 等价 free，高危坑：realloc 失败返回 NULL，不要直接赋给原指针，会丢失原地址！
```
```

if(p == NULL)
{
    return -1;
}
printf("malloc %p\n", p);
//p = (int*)realoc(p, 10);//重新分配10字节大小，删除原来后面90字节
```

指针类型看整体，指向类型看去掉 * 剩下的；
指向哪里看地址值；
指向类型决定指针 + 1 跳多远。

### 3.5 指针本身占据的内存区
